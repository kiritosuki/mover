使用时需要插入少量的mock数据
```sql
-- 插入车辆（空闲状态，位置在成都附近与现有POI匹配）
INSERT INTO vehicle (id, lon, lat, speed, update_time, status, tybe, size, capacity) VALUES
(1, 104.07, 30.55, 13.9, NOW(), 2, 1, 0, 5000),
(2, 104.05, 30.57, 11.1, NOW(), 2, 1, 0, 3000),
(3, 103.98, 30.50, 12.5, NOW(), 2, 2, 0, 4000);

-- 插入待处理订单（status=1）
-- POI 1,2 是 tybe=1（加油站），POI 3 是 tybe=2
-- cargo 1 是普通(tybe=1)，cargo 2 是危化品(tybe=2)
INSERT INTO shipment (id, start_poi_id, end_poi_id, create_time, update_time, status, cargo_id, count) VALUES
(1, 1, 3, NOW(), NOW(), 1, 1, 3),
(2, 2, 3, NOW(), NOW(), 1, 1, 5),
(3, 1, 3, NOW(), NOW(), 1, 2, 2);
```

整体业务流程：

```
┌─ 每 20 秒轮询一次 ─────────────────────────────────────────┐
│                                                             │
│  1. 查询待处理订单 (status=1)                                │
│              ↓                                              │
│  2. 对每个订单，并发执行 doCreateOrderTask:                    │
│              ↓                                              │
│     ┌─ NewDispatchContext ──────────────────┐               │
│     │ 一次性查询全局数据:                      │               │
│     │   - 所有车辆 (含运行中)                  │               │
│     │   - 所有订单                            │               │
│     │   - 所有货物类型 → 构建 CargoMap         │               │
│     └──────────────────────────────────────┘               │
│              ↓                                              │
│  3. 遍历空闲车辆，对每辆调用 EvaluateVehicle:                  │
│        a. 前置校验（类型匹配 + 容量够不够）                     │
│           → 不通过直接跳过                                    │
│        b. 构建快照（等待时间/空驶距离/运输距离等）                │
│        c. CalcSingleVehicleCost 计算该车代价                  │
│              ↓                                              │
│  4. 选 cost 最小的车辆                                       │
│              ↓                                              │
│  5. CollectAndPrintMetrics 打印全局指标报告                    │
│              ↓                                              │
│  6. 路径规划 → 创建任务 → 更新订单状态                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

会把相关指标的计算和每辆车的代价计算结果打印到控制台上
其中代价值为相对值，无单位，未作归一化
