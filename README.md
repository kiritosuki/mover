# 开发说明
剩下主要是两方面的内容，一个是数据统计，一个是cost的计算
开发之前可以先用tree命令让AI帮你分析一下目录结构，确认一下你的补充代码应该加在哪里
其实主要是 mover/internal/task 中的内容 在 order_task.go 中有详细说明

## 数据统计
主要参考上周课上要统计的数据，存在内存中即可，在某个地方声明全局变量
## cost 计算
我留出的代码块是创建task的部分，希望能根据task对每个订单选择局部最优的车辆，确定车辆以后再创建任务，所以这里cost的计算应该是前置条件，后续任务的执行，车辆的位置更新模拟逻辑应该是写完了，在 mover/internal/task 中

## debug技巧

- 我用zap库来打印日志，在 mover/internal/Logger 中有一个 Logger 全局变量，使用`Logger.Logger.Info(...)`就能打印日志了，或者`Warn / Error`
- 跑起来报错是因为有一个变量声明了但未使用，注释里有说明，计算cost可能会用到
- 数据库表删了很多，目前是最简单的雏型版本，你可能要自己拿AI mock一点数据测试，表变动太大之前的数据不能用了（之前的表设计的太复杂了，写起来晕头转向，我已疾苦）
- 需要补充一个`config-dev.yml`文件，我发在qq群里
- 可以让AI写代码，但注释尽量详细一点不然我看不懂（弱TvT
- 很多地方我都写了go程做多并发优化，用了最简单的互斥锁mutex，状态转换时可能涉及到对status的修改，并发条件下要加CAS锁，AI有时会忘
