# 车辆统计量

## 1. 模块设计目标
统计模块旨在为调度系统提供全局的运营概况。通过对车辆状态、订单流程和物理指标（距离、装卸）进行模块化建模，支持系统性能评估（如利用率、吞吐量）以及风险识别（如过长的等待时间、空驶浪费）。

## 2. 数据结构定义

### 2.1 统计核心 (`Statistics`)
包含五个子模块：车辆概况、利用率指标、时间指标、运输里程以及装卸指标。

### 2.2 具体指标说明
*   **VehicleOverview**:
    *   `TotalCount`: 平台注册的全部车辆数。
    *   `TypeCountMap`: 按车辆类型（如危化品、普通货车）统计的分布情况。
*   **UtilizationMetrics**:
    *   `ByVehicleCount`: `运行中车辆 / 总车辆`，反映车队整体活跃度。
    *   `ByCapacity`: `已用载重量 / 总载重量`，反映运力的深度利用情况。
*   **TimeMetrics** (对等待/运行时间的深度建模):
    *   提供最大值、最小值、平均值、中位数和标准差。
    *   `WaitTransRatio`: 等待时间与运输时间的比率，该值过高意味着车辆在调度环节存在大量 idle 损耗。
*   **DistanceMetrics**:
    *   包含空驶里程相关统计（总计、均值等）。
    *   `EmptyFullRatio`: 空驶里程 / 载重里程之比，核心的燃油/效率损耗指标。

## 3. 计算逻辑实现

### 3.1 车辆与利用率
通过遍历数据库中的 `vehicle` 记录，根据 `Status` 字段判断车辆是否处于作业状态，结合 `Capacity` 字段汇总总运力及实时运载力。

### 3.2 流程时间统计
基于 `order_task` 表中的 `CreateTime`（任务生成）与 `UpdateTime`（阶段流转）进行计算：
*   **等待时间**: `此刻 - CreateTime` (对于未完成任务) 或 `AcceptTime - CreateTime` (历史任务)。
*   **执行时间**: `Sequential` 处于运输阶段时的累计耗时。

### 3.3 模块化数学方法
引入 `computeBasicMetrics` 通用函数，支持任何数值序列的一键统计（包含排序抽中位数、平方差算标准差等逻辑）。

### 3.4 模块化统计项
代码中通过 `Statistics` 结构体进行了高度模块化划分，确保各指标计算互不干扰：
- **VehicleOverview**: 对车队静态属性的摸排。
- **UtilizationMetrics**: 动态运力监控。
- **TimeMetrics**: 流转效率分析。
- **DistanceMetrics**: 路径质量评估。
